### Gap-closure content contract (the actual load-bearing part of this UI-SPEC)

#### Gap 1 — Contacts legal/seller-identity block (`contacts.legalLine1-5`)

Replace the Russian entity data with a fixed 5-line TR seller-identity disclosure, one fact per
line (matches the existing 5-key/4-`<br/>` structure in `contacts/page.tsx:309-317` — **no code
restructuring needed**, content-only swap). Field order and labels:

| Line | Key | EN label + value pattern | TR label + value pattern |
|------|-----|---------------------------|---------------------------|
| 1 | `legalLine1` | `Legal name: [Placeholder]` | `Ünvan: [Placeholder]` |
| 2 | `legalLine2` | `Registered address: [Placeholder]` | `Yasal adres: [Placeholder]` |
| 3 | `legalLine3` | `Tax ID / MERSİS No: [Placeholder]` | `VKN / MERSİS No: [Placeholder]` |
| 4 | `legalLine4` | `IBAN: [Placeholder]` | `IBAN: [Placeholder]` |
| 5 | `legalLine5` | `Bank: [Placeholder]` | `Banka: [Placeholder]` |

**Pending-field convention (mandatory):** Use the literal string `[Placeholder]` for every value
that isn't yet known — this is not a new invention, it is the **already-established** convention
from Phase 5's legal pages (`messages/{en,tr}.json` → `legal.kvkk.s1Body`, `legal.mesafeli_satis.s1Body`,
etc. all currently ship `"[Placeholder]"` verbatim in both locales). Reusing this exact token means:
- The executor does not fabricate a fake TR address/VKN/IBAN (which would just be a second
  legally-material inaccuracy needing a second gap-closure pass before go-live).
- A future repo-wide `grep -rn "\[Placeholder\]"` (already a meaningful pre-go-live checklist item
  per `05-SECURITY.md`'s T-05-02) will also surface this block automatically — no separate tracking
  needed.
- **Never** re-ship a real-looking-but-fake RU or generic address/bank in its place — an empty/
  placeholder-marked field is honest; a fabricated one is not.

**EN/TR parity:** the labels (`Legal name:` / `Ünvan:` etc.) ARE translated per locale (that's the
whole point of having separate `en.json`/`tr.json` values) — only the `[Placeholder]` token itself
stays identical in both files, exactly like the Phase-5 legal-page precedent.

**Tone:** neutral, formal, legal-disclosure register — matches the existing 300-weight/10-14px
fine-print styling already in place. No marketing language, no exclamation points, no emoji.

**Behavior if a field is pending:** render `[Placeholder]` inline as part of the line (do NOT hide
the line, gray it out, or add a "coming soon" banner) — this matches how Phase 5's legal pages
already render pending sections (plain literal text, no conditional UI). Consistency with that
existing pattern is more important than inventing a bespoke "pending" treatment for one block.

#### Gap 2 — Footer copyright line (`Footer.tsx:214`)

Replace the RU-domain text with a neutral brand line, format:

```
© {year} American Creator
```

- **Drop the domain entirely** — do not substitute a different domain (e.g. a placeholder `.tr`
  domain); "American Creator" (brand name only, no URL) is the neutral, deploy-agnostic choice,
  consistent with how the Instagram social icon already links to the generic `instagram.com/` (no
  specific handle) rather than a fake handle.
- **Year:** use `{new Date().getFullYear()}` (dynamic) — **DETERMINED, not a coin-flip.** This is
  the single governed choice: a one-line change that prevents this exact class of gap (a stale
  hardcoded value) from recurring. Compute it inline in the JSX; it is SSR/hydration-safe (the year
  is stable within any single render, so server and client render the same value). Do NOT ship a
  hardcoded `2026`.
- **i18n:** keep this as a static JSX string, NOT a new `t()` i18n key. "American Creator" is a
  proper noun (same on EN/TR) and the existing code already hardcodes a proper noun in the same
  file without translation (`alt="American Creator"` on the logo `<img>`, `Footer.tsx:101,169`) —
  match that established precedent rather than adding a new `common.copyright` key for a single
  static string.
- **Markup:** keep the existing `&copy;` HTML entity (do not switch to the literal `©` Unicode
  glyph) — purely to minimize the diff; either renders identically.
- **Placement/style:** unchanged — same `<Typography sx={{ fontSize: 16, color:
  palette.primaryLight }}>` wrapper, same position (bottom-left of the footer's bottom row,
  opposite the payment icons). Do not resize, recolor, or reposition.
