## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next proxy → ARM BFF | `/auth/me/profile`, `/change-password`, `/export`, `/delete-account` требуют валидный JWT; необратимые действия (delete) — повторная аутентификация паролем |
| Blob download | Экспорт PII собирается BFF и скачивается клиентом локально (`<a download>`) — данные не покидают браузер пользователя |
| createOrder Authorization | Токен из localStorage уходит в исходящий fetch; прокси форвардит его → ARM привязывает заказ к customer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Elevation of Privilege | POST /auth/me/change-password (session fixation после смены пароля, FBG-22) | mitigate | BFF: `tokens_valid_after`-watermark инвалидирует пред-сменные токены; клиент после успешной смены делает `signOut()` + требует повторного входа свежим токеном. |
| T-03-11 | Elevation of Privilege | GET /auth/me/export, POST /auth/me/delete-account (доступ без авторизации) | mitigate | Оба вызова с `bearerHeader()`; BFF `requireCustomer()` верифицирует JWT на обоих эндпоинтах. |
| T-03-12 | Tampering | POST /auth/me/delete-account (необратимое удаление чужого/случайное) | mitigate | Обязательное подтверждение паролем в UI-диалоге (re-auth) + BFF верифицирует пароль; анонимизация, не hard-delete (D-09). |
| T-03-13 | Information Disclosure | Экспорт PII (GDPR-блоб) | accept | Данные принадлежат самому пользователю; скачиваются в его браузер (`<a download>`), не логируются и не шарятся; сборка PII — на BFF (requireCustomer). |
| T-03-14 | Spoofing | createOrder с Authorization (привязка к чужому аккаунту) | transfer | Привязку выполняет ARM по customer из JWT (не по полю из тела); клиент лишь прикладывает свой Bearer; гость без токена — без привязки. |
| T-03-SC | Tampering | npm/pip/cargo installs | n/a | Новых пакетов нет (03-RESEARCH §Package Legitimacy Audit) — supply-chain-гейт не применяется. |
</threat_model>
