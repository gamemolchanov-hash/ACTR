## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next proxy → ARM BFF | Запросы к `/auth/me/*` требуют валидный JWT (Authorization: Bearer) — прокси форвардит заголовок из входящего запроса |
| CSR auth-guard | `/account/*` страницы защищены клиентским `useEffect`-редиректом; реальная авторизация — на BFF |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Elevation of Privilege | GET /auth/me/orders, /auth/me/addresses (доступ без авторизации) | mitigate | Все protected-вызовы шлют `bearerHeader()` (Pitfall 5); BFF `requireCustomer()` верифицирует JWT на каждом вызове; CSR-guard редиректит на /login при отсутствии customer. |
| T-03-07 | Information Disclosure | Чужие заказы/адреса (IDOR) | transfer | BFF скоупит выборку к customer из JWT (`requireCustomer()`); фронт не передаёт customer_id — сервер берёт его из токена. |
| T-03-08 | Tampering | DELETE /auth/me/addresses/:id (удаление чужого адреса) | transfer | BFF проверяет принадлежность адреса customer из JWT перед удалением; клиент шлёт только id + Bearer. |
| T-03-09 | Information Disclosure | PII заказов/адресов в клиентском state | accept | Данные принадлежат самому пользователю (его сессия); рендер только в его браузере; без шаринга/логирования PII на клиенте. |
| T-03-SC | Tampering | npm/pip/cargo installs | n/a | Новых пакетов нет (03-RESEARCH §Package Legitimacy Audit) — supply-chain-гейт не применяется. |
</threat_model>
