## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next proxy | Недоверенный ввод форм (register/login/forgot/reset) пересекает границу; прокси server-side инъектит X-Storefront-Key |
| Next proxy → ARM BFF | BFF владеет всей auth-логикой (bcrypt, rate-limit, JWT RS256), валидирует zod-схемами |
| localStorage ↔ JS | `arm_token` хранится в localStorage — доступен любому JS на странице (XSS-поверхность) |

| T-03-02 | Information Disclosure | POST /auth/register (account enumeration) | mitigate | BFF отдаёт единое сообщение для email-taken/дубликата; клиент показывает ответ BFF как есть, не различает «занято/новое». |
